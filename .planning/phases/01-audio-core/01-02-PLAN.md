    **Add import at top:**
    ```rust
    use makepad_widgets::*;
    use std::sync::Arc;
    use crate::audio::{AudioParams, init_audio_async};
    ```

    **Extend App struct** — add two fields after `ui: WidgetRef`:
    ```rust
    #[derive(Live)]
    pub struct App {
        #[live]
        ui: WidgetRef,
        // Audio — initialized in after_new_from_doc, held for program lifetime.
        // _stream: Drop silences audio — must NOT go out of scope.
        // _audio_params: Phase 3 will write pan values to these Shared atomics.
        #[rust]
        _stream: Option<cpal::Stream>,
        #[rust]
        _audio_params: Option<Arc<AudioParams>>,
    }
    ```
    IMPORTANT: Use `#[rust]` attribute (not `#[live]`) for non-Makepad fields. Use `Option<T>` because `#[derive(Live)]` requires a default — `None` is the default before after_new_from_doc fires. Remove `LiveHook` from the derive list — we implement it manually below.

    **Implement LiveHook manually** (audio init goes here):
    ```rust
    impl LiveHook for App {
        fn after_new_from_doc(&mut self, _cx: &mut Cx) {
            let (stream, params) = init_audio_async();
            self._stream = Some(stream);
            self._audio_params = Some(Arc::new(params));
        }
    }
    ```

    **Keep LiveRegister and AppMain unchanged:**
    ```rust
    impl LiveRegister for App {
        fn live_register(cx: &mut Cx) {
            makepad_widgets::live_design(cx);
        }
    }

    impl AppMain for App {
        fn handle_event(&mut self, cx: &mut Cx, event: &Event) {
            self.ui.handle_event(cx, event, &mut Scope::empty());
        }
    }

    app_main!(App);
    ```

    **Keep live_design! block unchanged** — dark window, 900x600, body View.

    After writing, run `cargo check --bin ghostinstrument` to confirm it compiles before moving to cross-compile.
