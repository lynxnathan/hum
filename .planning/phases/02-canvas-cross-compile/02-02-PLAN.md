**Step 2 — app.rs LiveRegister:** In `impl LiveRegister for App`, add `crate::canvas::live_design(cx);` AFTER `makepad_widgets::live_design(cx)`. Per RESEARCH.md Pitfall 1, makepad_widgets must be registered first.

Before (app.rs):
```rust
impl LiveRegister for App {
    fn live_register(cx: &mut Cx) {
        makepad_widgets::live_design(cx);
    }
}
```

After:
```rust
impl LiveRegister for App {
    fn live_register(cx: &mut Cx) {
        makepad_widgets::live_design(cx);
        crate::canvas::live_design(cx);
    }
}
```

**Step 3 — app.rs live_design!:** Replace the `<View>` body placeholder with `<CanvasWidget>`. The App's `live_design!` block must use `link::widgets::*` (already present) — add `use link::shaders::*;` ONLY if needed for the CanvasWidget reference (it likely is not needed in app.rs since canvas.rs declares the shader).

Before (app.rs live_design!):
```rust
body = <View> {
    width: Fill
    height: Fill
}
```

After:
```rust
body = <CanvasWidget> {}
```

The `width: Fill, height: Fill` walk properties default from the widget's own `live_design!` declaration in canvas.rs (`pub CanvasWidget = {{CanvasWidget}} { width: Fill, height: Fill ... }`), so no need to repeat them in app.rs.
